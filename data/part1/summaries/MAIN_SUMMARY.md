### **System Overview**
"Room Check" is an application designed to collect environmental sensor readings (such as CO2, PM2.5, temperature, humidity, and VOCs), evaluate them against safety thresholds, detect operational system issues, and generate alerts. 

### **Core Components & Data Pipeline**

**1. Data Ingestion (Inputs)**
The system can pull data from a variety of hardware and manual sources, normalizing everything into a standard schema.
* **`core/inputs.py`:** The primary translation layer that collects data, removes impossible values, and adds necessary context metadata (like building schedules and timestamps).
* **`core/inkbird.py` & `core/inkbird_manual.py`:** These modules handle data from Inkbird monitors. The system can read Bluetooth (BLE) advertisement payloads automatically. If Bluetooth is unreliable, a guided manual entry flow prompts the user to manually type the values displayed on their device.
* **`core/dyson_pull.py`:** Handles network-connected Dyson Pure Cool / Hot+Cool devices. It manages a one-time cloud login flow, connects to the device locally, pulls live air-quality data, and converts metrics to match the Room Check schema.

**2. Evaluation & Logic**
* **`core/room_check.py`:** This is the logic engine of the system. It compares the gathered readings against threshold specifications to assign per-metric and overall room statuses (e.g., GREEN, YELLOW, RED). It is also responsible for checking "operational drift"—detecting if building systems (like HVAC) are failing to behave as expected, which could waste money or pose risks.

**3. Alerts & Reporting**
* **`core/alerts.py`:** Takes the statuses generated by the evaluation phase and turns them into human-readable, plain-language safety alerts so building staff know what actions to take. Additionally, it calculates a **Community Resilience Index** (a score from 0–100) that combines the live sensor conditions with known building vulnerabilities to show how prepared or at-risk the building is.

**4. Orchestration**
* **`core/main.py`:** The central hub of the software. It orchestrates the full flow: fetching inputs, evaluating the room, detecting drift, and creating alerts. It also handles the system's outputs, which include printing console reports, backing up data to local CSV files, and optionally posting data to a cloud dashboard.

### **User Accessibility**
* **`student_quickstart.md`:** To make the tool easier to use without memorizing complex command-line flags, the system includes an interactive "quickstart" launcher designed for students. It uses console prompts to ask users what hardware they are using and how they want to run the program, automatically executing the underlying scripts on their behalf.

The application monitors environmental safety and building efficiency through several specialized modules:

* **Orchestration (`main.py` & `student_quickstart.py`):** * `main.py` is the central hub that runs the full program loop: gathering data, evaluating the room, detecting drift, and creating alerts. 
  * `student_quickstart.py` provides an interactive, menu-driven launcher so students can run the system without memorizing command-line flags.
